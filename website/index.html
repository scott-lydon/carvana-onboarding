Carvana Onboarding Recovery Layer

A graceful-degradation, OCR-augmented layer that sits between Carvana's sell-flow entry form and their existing vendors. It separates the six distinct failure modes Carvana currently collapses into one hostile error message, and routes each to its correct next-action prompt.

The problem we are fixing

Carvana today

  • Six failure causes (bot detection, vendor coverage miss, transient backend error, format edge case, character ambiguity, foreign plate) all render the same string: "we couldn't find that license plate."
  • User cannot tell which mode hit them, cannot fix it, leaves for a competitor.
  • VIN tab submission silently resets to license plate on failure with no error message, contradicting the "VIN OR plate" promise.
  • Texas plates with the asterisk separator fail 100% (5 for 5 in our corpus).
  • Bot detection misfires on legitimate automation users (developers, accessibility users, VPN users) with the same blame-the-user copy.
  • No OCR fallback. No character-permutation recovery. No format normalization.

Our recovery layer

  • Cascade lookup: primary vendor → fallback vendor → OCR camera capture.
  • Format normalization strips Texas asterisks, whitespace, dashes, dots; uppercases; trims.
  • Character-permutation recovery suggests I↔1, O↔0, B↔8, S↔5, Z↔2 on lookup misses.
  • Discriminated-union return shape: resolved, not_found, transient_error, bot_detected, format_error. Each renders its own copy and next-action.
  • Form and tab state preserved across every error. No silent resets.
  • SMS resume link surfaces at every dead-end. Proactive chat after 3 failures.
  • Telemetry per failure mode so the recovery rate can be measured for the first time.

Topology

Drop-in gateway pattern. Carvana's existing frontend posts to our gateway instead of their direct vendor call; everything downstream is plug-compatible with their current stack. The gateway owns the cascade, the timeouts, and the failure-mode discrimination.

Components

VendorCascade

Pure domain module. Tries primary vendor with a 2s timeout, falls through to fallback on miss/error, returns a discriminated-union result.

Pros

  • Vendor-agnostic; swap adapters in one file.
  • Bounded worst-case latency (sum of timeouts).
  • Structured return = no shared error string.

Cons

  • Two-vendor minimum doubles integration surface.
  • Mocked tests forbidden by constitution (real sandboxes only).

DegradationLayer

React context wrapping the entry form. Pattern-matches the cascade's return shape to user-facing copy + next-action prompt. The load-bearing component.

Pros

  • One named copy path per failure cause.
  • Tab and field state preserved on error.
  • Honest "we couldn't reach our data" copy, not blame-the-user.

Cons

  • More copy to maintain than a single generic message.
  • Bot-detection differentiated copy is a Trust & Safety decision Carvana has to greenlight.

OCRCapture

Browser getUserMedia for camera capture; ships a cropped image to /api/ocr/recognize which calls Google Cloud Vision. iOS variant uses Apple Vision on-device for the privacy story.

Pros

  • Recovers users where data coverage genuinely misses.
  • iOS on-device = no PII over the wire.
  • ~$1.50 per 1k web-side OCR calls. Trivial at scale.

Cons

  • Camera permission is its own UX speed bump.
  • Multi-color plates (Hawaii rainbow, fade specialty plates) need confidence-threshold confirmation.

NormalizationLayer

First line of defense against the Texas asterisk bug. Strips non-alphanumeric characters, uppercases, trims whitespace, shows the user what was stripped as a trust signal.

Pros

  • Fixes EC1 (Texas asterisk) for an entire state.
  • Fixes EC4 (whitespace, dashes, dots) silently.
  • One-day eng change.

Cons

  • Aggressive stripping could mask legitimate special characters on diplomatic / antique plates. Mitigation: per-state allowlists in slice 2.

EventReporter

Each named failure mode is a stable event name. Logs to a local SQLite for the demo; would wire to Carvana's existing telemetry in production.

Pros

  • First time Carvana can measure per-mode failure rates.
  • Recovery dollar estimate moves from inference to observation.
  • Adversary-replayable test fixtures.

Cons

  • Event-name drift is a real risk (CAT-10 in QA_ADVERSARY.md); enforced via TS union.

ConsentManager

Buy-side TCPA-compliant consent UI. SMS opt-in defaults OFF; User Agreement, Privacy Policy, E-SIGN consent surface as separate checkboxes, not click-wrap.

Pros

  • Literal opposite of Carvana's current B5 default-on toggle.
  • Audit-trail logging at /api/consent/log.
  • Reduces TCPA exposure.

Cons

  • Higher opt-in friction = fewer marketing-eligible users (intentional, ethically defensible).

Data flow (sell-side cascade-fallback path)

The user enters a plate that the primary vendor does not have. The cascade falls through to the fallback vendor, resolves, returns. The DegradationLayer surfaces a transparency note ("we used our backup data source"). User sees their car and an estimated offer in under 4 seconds.

Trade-offs we accept

Single Render web service in the prototype

Why we accept it: One URL, zero CORS, deployable in 5 minutes by a Carvana PM via blueprint launch.

When it would bite: A backend bug can take down the static-serve path. Production Carvana splits this into a CDN-fronted static site + standalone API.

VinAudit as prototype primary instead of Carfax

Why we accept it: Self-service signup. Carfax QuickVIN+ requires dealer sales contract (weeks of friction).

When it would bite: VinAudit's coverage profile may differ from Carfax's. The adapter interface makes the swap a one-file change for production.

Honest bot-detection messaging

Why we accept it: Sliver of obscurity loss against attackers in exchange for meaningful legitimate-user recovery. Sophisticated attackers detect the block independently anyway.

When it would bite: A specific abuse vector that DEPENDS on attackers not knowing detection fired. We have not identified one.

Demo-only mock for prequal soft-pull

Why we accept it: Real soft-pull requires Equifax/TransUnion/Experian integration with PII flowing. Out of scope for a one-week prototype.

When it would bite: If the demo gets confused for production. Mitigation: explicit "demo data" label in the UI.

Cost analysis

Recovery vs. implementation cost (annual)

Per-call cost (log scale)

Cloud Vision OCR is the only meaningful variable. At Carvana's volume the per-call costs are dominated by vendor lookup fees, which Carvana already pays under their dealer contracts.

Trust boundaries

PII never enters logs in raw form. The DPPA boundary is enforced at the type-level interface, not by runtime checks. On-device OCR keeps image bytes off the network on iOS.
