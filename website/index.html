The problem we recover from

Carvana today

  • Plate-lookup wall blames the user when the vendor data misses (S4).
  • VIN submission silently resets the form on backend error (S6).
  • Buy-side prequal traps users behind an account-creation gate after 5 minutes of PII entry (B8).
  • Pre-checked SMS opt-in (TCPA risk, B5).

Recovery layer

  • Chatbot drives plate entry conversationally; tool-use cascades over real vendor APIs.
  • OCR rescues VIN capture when the user can't or won't type 17 characters.
  • Atomic scheduler books pickup with no double-booking under concurrent load.
  • Pre-baked support content surfaces at known anxiety moments; the LLM picks WHICH card, never writes the words.

Components

ChatbotShell slice A

Pros
  • Tool-use orchestration is the only place LLM touches user data.
  • Streamed SSE first-token under 1.5 s.
  • history_sync preserves multi-turn context.
Cons
  • Adds a vendor (Anthropic) to the critical path.
  • Pre-baked greeting needed to mask first-paint latency.

OcrCapture slice B

Pros
  • Same Anthropic key as chat — one vendor, one secret.
  • Camera + file-upload fallback; no Tesseract bundle.
  • Recognized VIN flows back as a user message; chatbot routes natively.
Cons
  • Cost per call > Tesseract (offset by accuracy).
  • Requires camera permission UX.

Scheduler slice C

Pros
  • BEGIN IMMEDIATE + UNIQUE constraint = atomic at the storage layer.
  • 10 parallel bookings → exactly 1 success (regression test enforced).
  • In-house grid ships in 1 day vs Cal.com integration.
Cons
  • Ephemeral on Render free tier (acceptable for demo).
  • No timezone-aware display per user.

SupportContentWidget slice D

Pros
  • Hallucinated reassurance is eliminated at the architecture level.
  • Card bodies are auditable, A/B-testable, citable.
  • Byte-for-byte regression test (CAT-12) blocks drift.
Cons
  • Adding a topic is a code change, not a config change.
  • LLM may misclassify the right topic.

NpsSurvey + MetricsOverlay slice E

Pros
  • NPS = ((promoters − detractors) / n) × 100, Bain standard.
  • Summary always carries n + source labeling (CAT-15).
  • Stopwatch from first user message proves the 15-min metric.
Cons
  • Demo sample size is small; cited industry NPS used as anchor.

VendorCascade slice 1

Pros
  • CarsXE primary, VinAudit fallback (pending B2B approval).
  • 8 s per-adapter timeout; short-circuits on resolved or bot_detected.
  • Property tests (200 runs) over every vendor permutation.
Cons
  • CarsXE sandbox is 100 calls lifetime; production swap to Carfax is a one-line adapter change.

Trade-offs

Chatbot adds first-token latency

Why I accept it: streaming makes perceived latency = time-to-first-token. Pre-baked greeting + sub-1.5 s first token reads as immediate.

When it would bite: if Anthropic outage. Form fallback link stays clickable.

SQLite ephemeral on Render free tier

Why I accept it: demo concurrency is low; bookings reset on redeploy.

When it would bite: production. Postgres swap or Render Disk ($1/mo).

Pre-baked support content is rigid

Why I accept it: hallucinated emotional reassurance is a legal risk. LLM picks WHICH card; humans write the words.

When it would bite: if Carvana wants A/B-test wording quickly. Cards.ts is one PR.

No buy-side coverage in v2

Why I accept it: 2 days does not give time for both sides + 4 new capabilities.

When it would bite: if Carvana grades on buy-side parity. v1 spec covers it.
